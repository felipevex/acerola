Acerola Docs

class AcerolaServerService

package acerola.server.service

extended by AcerolaServerServiceRest

Classe base para serviços do servidor Acerola que gerencia requisições HTTP, validação de dados e comportamentos.

Responsabilidades:

  • Processamento de requisições: Gerencia o ciclo de vida de uma requisição HTTP no servidor Acerola.
  • Validação de dados: Valida estruturas de dados de entrada como parâmetros e corpo da requisição.
  • Gerenciamento de resposta: Prepara e envia respostas HTTP adequadas com os dados processados.
  • Extensibilidade: Fornece uma estrutura para implementações específicas através de métodos que podem ser sobrescritos.

Constructor

new(req:AcerolaServerRequestData, res:AcerolaServerResponseData)

Cria uma nova instância do serviço inicializando com os dados de requisição e resposta. Também inicializa o gerenciador de comportamentos e chama o método de configuração.

Parameters:

req:AcerolaServerRequestData

Dados da requisição HTTP recebida

res:AcerolaServerResponseData

Objeto para preparar a resposta HTTP

Methods

run():Void

Método principal que executa a lógica do serviço. Este método deve ser obrigatoriamente sobrescrito por subclasses.

Throws:

String

Erro indicando que o método deve ser sobrescrito

runInfo():Void

Executa uma função informativa que retorna metadados sobre a requisição atual. Útil para fins de depuração ou para endpoints de informação.

runTimeout():Void

Método executado quando ocorre um timeout na requisição. Pode ser sobrescrito por subclasses para tratamento específico de timeout.

setup():Void

Realiza configurações adicionais para o serviço. Este método deve ser sobrescrito por subclasses para configurações específicas.

setupBehavior():Void

Configura comportamentos específicos para este serviço. Este método deve ser sobrescrito por subclasses para adicionar comportamentos personalizados.

validate():Void

Executa os processos de validação de corpo e parâmetros da requisição. Chama internamente os métodos validateBody e validateParams.

validateBody():Void

Valida o corpo da requisição utilizando o validador configurado.

Throws:

AcerolaServerError

Se o corpo da requisição não passar na validação

validateParams():Void

Valida os parâmetros da requisição utilizando o validador configurado.

Throws:

AcerolaServerError

Se os parâmetros não passarem na validação
