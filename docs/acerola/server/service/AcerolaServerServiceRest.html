class AcerolaServerServiceRest<S>

package acerola.server.service

extends AcerolaServerService

extended by AcerolaServerDatabaseService

Classe responsável por processar requisições REST e gerar respostas apropriadas com conteúdo estruturado. Esta classe serve como base para implementação de serviços REST no framework Acerola, permitindo o processamento de requisições HTTP e a geração de respostas padronizadas.

Para utilizar esta classe, você deve criar uma subclasse específica para cada endpoint da sua API, sobrescrevendo pelo menos o método run() para implementar a lógica de negócio do serviço.

Exemplo de uso básico:

// Serviço que retorna um JSON simples
class HelloWorldJsonService extends AcerolaServerServiceRest<{hello:String}> {
    override function run() {
        this.resultSuccess({hello: "world"});
    }
}

// Serviço que retorna conteúdo HTML
class HelloWorldTextService extends AcerolaServerServiceRest<Dynamic> {
    override function run() {
        this.resultHtml('<div>hello world</div>');
    }
}

Para validação de parâmetros de requisição, você pode sobrescrever o método setup():

class TestGetService extends AcerolaServerServiceRest<TestGetServiceData> {
    override function setup() {
        super.setup();
        this.paramsValidator = TestGetServiceDataValidator;
    }
    
    override function run() {
        var params:TestGetServiceData = this.req.params;
        this.resultSuccess(params);
    }
}

Responsabilidades:

  • Processamento de Requisições: Gerencia o fluxo de processamento de uma requisição REST, incluindo validação e execução.
  • Gerenciamento de Comportamentos: Executa comportamentos em ordem sequencial antes de processar a requisição principal.
  • Formatação de Respostas: Converte dados em respostas HTTP apropriadas com diferentes formatos de conteúdo.
  • Tratamento de Erros: Captura e processa exceções, convertendo-as em respostas de erro padronizadas.

Construtor da classe que inicializa o serviço com os dados da requisição e resposta. Este método configura o ambiente básico para o processamento da requisição, incluindo a inicialização de comportamentos e tratamento de exceções iniciais.

Parameters:

os dados da requisição HTTP recebida

res:AcerolaServerResponseData

o objeto para preparar a resposta HTTP

Methods

resultError(error:AcerolaServerError):Void

Retorna uma resposta de erro ao cliente com base no objeto de erro fornecido. O status HTTP e o conteúdo são extraídos do objeto de erro. Este método é utilizado automaticamente pelo tratamento de exceções interno, mas também pode ser chamado manualmente para casos de erro específicos.

Parameters:

error:AcerolaServerError

o objeto de erro que contém informações sobre o erro ocorrido

@:value({ status : 200 })resultHtml(data:String, status:Int = 200):Void

Retorna uma resposta HTML ao cliente com o conteúdo fornecido. Este método é útil quando você precisa retornar uma página HTML completa ou qualquer conteúdo textual com formatação HTML.

Parameters:

data:String

o conteúdo HTML a ser enviado na resposta

status:Int

o código de status HTTP da resposta

@:value({ status : 200 })resultSuccess(data:S, status:Int = 200):Void

Retorna uma resposta JSON de sucesso ao cliente com os dados fornecidos. Este é o método mais comum para APIs REST, permitindo retornar dados estruturados que serão automaticamente serializados para JSON.

Parameters:

data:S

os dados a serem serializados como JSON na resposta

status:Int

o código de status HTTP da resposta
